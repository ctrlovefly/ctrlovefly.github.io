Optimization Capsule
2019-10-25

第一次搭建个人博客

凌晨00:04，我坐在桌前准备开始复习多元统计分析。但是白天看过的David Abel的学术会议Notes仍在我的脑海中浮现，更为印象深刻的是，进到他的个人主页以后，简洁有趣的风格更令人难以释怀。一想到自己早就想要创建个人主页的计划一拖再拖，就不能忍耐，就是现在，立刻行动吧。

我参考的是b站博主CodeSheep做的利用Hexo框架和GithubPages搭建的视频教程。他做的是linux系统下的示范，尽管windows下指令差不多，但还是走了些弯路，在这总结下我在windows10下利用Hexo和GithubPages搭建免费个人博客的步骤。也算是纪念吧，今天正好是1024~

Outline：

  • 下载Note.js和Git并安装。

  • 使用命令行调用NPM(or cnpm)下载Hexo。

  • 初始化Hexo，在本地端口可打开。

  • 更换并修改网站主题，修改网站链接地址。

Procedure：

  1. 在提纲中的网址中下载这两个东西并安装，安装过程全程下一步即可。

    Node.js 就是运行在服务端的 JavaScript

    NPM是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题，常见的使用场景有以下几种：

    • 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
    • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
    • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

    Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

    其中，我们需要Node.js中的NPM包管理工具，下载Hexo。Git指令在远程部署网站到githubpages的时候需要使用。以上的说明来自菜鸟教程。

    Git & Node.js

  2. 打开win的命令行（win+s打开搜索框，搜索cmd打开）：

    • 检验软件版本以确定安装

      若出现版本号，则已安装完成。

    • 切换至淘宝镜像下载Hexo

    成功安装

  3. 初始化Hexo

    • hexo -v检查一下hexo现在的版本号，若出现版本号，则可开始初始化一个博客。

    • 新建一个blog文件夹（之后所有的文件都在里面），切换目录到该文件夹（cd 地址/换盘时直接E:/查看文件夹下所有文件dir/打开某个文件直接写在该目录下写文件名），如下图：

    • hexo init初始化一个博客到切换后的目录下，这时再dir一下发现目录下多出了许多文件，就是网页相关的文件。

    • ​ hexo server在本地端口访问网页（ctrl+C退出）。

​ 打开浏览器访问端口http://localhost:4000就可以看到一个原始的博客啦！
